## Teacher's Guide

**Course:** Term 1: Hello Python!

## 📋 Lesson Overview

### Purpose

This is the culminating celebration of Term 1. The primary goals are to: (1) give every student a moment to shine, (2) celebrate individual and collective achievements, (3) build confidence through public presentation, and (4) generate excitement for Term 2. This lesson is about celebration and closure, not new content.

### Learning Objectives

By the end of this lesson, students will have:

1. Presented their completed project to peers
2. Practiced public speaking and presentation skills
3. Celebrated their own and classmates' achievements
4. Reflected on their Term 1 learning journey
5. Built excitement for Term 2

### Prerequisites

- Students should have completed Joke Machine project (Week 7)
- All projects should be submitted/accessible before class
- Any students who didn't complete should have a plan (see Differentiation)

### Materials Needed

- Computer with internet access (for displaying student projects)
- Projector/screen share capability
- Certificates (printed or digital)
- Awards/recognition materials (optional)
- Timer for presentations
- Celebration items (optional: stickers, treats, decorations)
- Camera for class photo (optional)
- Backup plan for technical issues

### Pre-Lesson Preparation

1. **Test all student projects** - Verify links work before class
2. **Prepare presentation order** - Mix skill levels, volunteers first
3. **Print certificates** - Personalized for each student
4. **Prepare awards** - Decide on categories and recipients
5. **Set up celebration elements** - Decorations, treats if applicable
6. **Create Term 2 preview** - Exciting snippets to share
7. **Prepare backup projects** - In case of technical failures

## 📚 Detailed Teaching Guide

### Part 1: Welcome & Energy Setting (2 minutes)

#### Goals

- Create exciting, celebratory atmosphere
- Set positive tone for presentations
- Reduce nervousness through collective excitement

#### Script/Talking Points

> "Welcome to the TERM 1 SHOWCASE PARTY! 🎉
>
> This is it—the moment you've been working toward for 8 weeks! Today is YOUR day to shine!
>
> You've learned SO much. You've built REAL programs. And now you get to show them off!
>
> A few things before we start:
>
> - Everyone here has accomplished something amazing
> - We're all here to support each other
> - We WILL laugh at the jokes—that's the whole point!
> - After each presentation, we clap and cheer!
>
> Are you ready to celebrate?!"

**Get an enthusiastic response!**

---

### Part 2: Final Preparations (8 minutes)

#### Goals

- Give students last chance to fix issues
- Reduce anxiety through preparation
- Handle any technical problems

#### Setup

> "You have 8 minutes to:
>
> 1. Test your code ONE more time
> 2. Practice what you'll say
> 3. Take some deep breaths!
>
> I'll come around to check on everyone. If you have any last-second issues, raise your hand."

#### Teacher Actions During Prep

1. **Circulate quickly** - Check each student's readiness
2. **Identify issues** - Note any projects that won't run
3. **Quick fixes only** - No major debugging now
4. **Reassure nervous students** - "You've got this!"
5. **Confirm presentation order** - Finalize the list

#### Handling Non-Complete Projects

If a student didn't finish:

- Have them present what they DO have
- Focus on what they learned, not what's missing
- Consider pairing with a complete project to "co-present"
- NEVER shame or exclude—everyone presents something

#### 2-Minute Warning

> "Two more minutes! Make sure your project is ready to run. Remember, you'll introduce yourself, run your code, and share one thing you're proud of. That's it! Easy!"

---

### Part 3: Showcase Presentations (30 minutes)

#### Goals

- Every student presents successfully
- Maintain supportive, celebratory atmosphere
- Keep presentations moving on time

#### Presentation Structure

Each presentation should follow this format (~2-3 minutes):

1. **Introduction (30 sec)**
   - Student name
   - Project name (if they gave it one)
2. **Run the Project (1-1.5 min)**
   - Display the code running
   - Let jokes land naturally
3. **Reflection (30 sec)**
   - One thing they're proud of
   - OR one thing they learned
4. **Applause (15 sec)**
   - Lead enthusiastic applause
   - Positive comment from teacher

#### Time Management

With 15-20 students, you have ~2 minutes each.

| Class Size | Time Per Student      | Strategy                |
| ---------- | --------------------- | ----------------------- |
| 10-12      | 2.5-3 min             | Full presentations      |
| 13-18      | 2 min                 | Streamlined format      |
| 19-25      | 1.5 min               | Quick showcase style    |
| 25+        | Consider small groups | Split into two sessions |

**Use a timer!** Gently signal when time is running low.

#### Leading Applause

After EVERY presentation:

- Start clapping immediately
- Make eye contact with the presenter
- Say something specific and positive:
  - "Great job, [name]! I loved the [specific detail]!"
  - "That was hilarious! The [joke] really got me!"
  - "Awesome work! Your [decorations/code/creativity] really stood out!"

#### Handling Technical Issues

| Issue              | Quick Solution                                                |
| ------------------ | ------------------------------------------------------------- |
| Project won't load | Have student describe what it does while you troubleshoot     |
| Trinket is slow    | Fill time with questions about the project                    |
| Code has bug       | "This is a great debugging opportunity! What might be wrong?" |
| Student freezes    | Gently prompt: "Tell us about one of your jokes"              |
| Audio issues       | Read the output aloud for the class                           |

#### Presentation Order Strategies

**Option 1: Volunteers First**

- Ask for volunteers initially
- Nervous students see examples first
- Can get stuck if no one volunteers

**Option 2: Random Order**

- Draw names from a hat
- Fair and unpredictable
- Can be anxiety-inducing

**Option 3: Strategic Order**

- Confident student first (sets good example)
- Mix skill levels throughout
- Save a strong one for the end
- **(Recommended approach)**

**Option 4: Alphabetical**

- Predictable
- May not be optimal order
- Simple to manage

#### Goals

- Recognize EVERY student's achievement

| Week | Topic          | What You Can Now Do                    |

>

> "Take 30 seconds to think about each question..."

1. "What was your FAVORITE thing you learned?"
2. "What are you MOST proud of?"
3. "What do you want to learn next?"

**Optional: Have 2-3 students share briefly.**

#### Term 2 Preview (3 minutes)

> "Now let me give you a SNEAK PEEK at what's coming in Term 2..."

**Build excitement with examples:**

```python
name = input("What's your name? ")
print("Hello, " + name + "!")
```

> "In Term 2, your programs will TALK to the user! They can type responses, and your program reacts!"

```python
age = 10
if age >= 13:
    print("You're a teenager!")
else:
    print("You're not a teen yet!")
```

> "Your programs will make DECISIONS! They can choose different paths based on information!"

**Loops:**

```python
for i in range(5):
    print("Hello!", i)
```

> "You'll make code REPEAT automatically! No more copying and pasting the same thing!"

**Projects Coming:**

> "You'll build quiz games, fortune tellers, choose-your-own-adventures, number guessing games, and maybe even chatbots!"

> "If you thought Term 1 was fun, Term 2 is going to blow your mind! Get ready!"

## 🎯 Assessment & Differentiation

### Assessment Focus

This lesson is about **celebration, not evaluation**. However, note:

**Observation Points:**

- Presentation confidence level
- Ability to explain their project
- Peer support behaviors
- Engagement during others' presentations

**Use for Term 2 Planning:**

- Which students need more support?
- Which are ready for challenges?
- Which topics need review?

### Differentiation Strategies

#### For Students Who Didn't Complete

**NEVER exclude a student from presenting.**

Options:

1. Present what they have (even if incomplete)
2. Describe what they TRIED to build
3. Share what they learned from the attempt
4. "Co-present" with a partner who completed

**Script for incomplete project:**

> "Even though [name]'s project isn't quite finished, they learned SO much about [concepts]. Tell us about what you worked on..."

#### For Nervous Presenters

- Offer to stand with them
- Allow seated presentation if standing is too scary
- Reduce requirements (just run code, no explanation needed)
- Present earlier in the order (less buildup anxiety)
- Practice privately before showcase starts

#### For Overconfident Students

- Channel energy positively
- Give them a "helper" role for nervous students
- Ensure they're respectful during others' presentations
- Consider last spot (big finale energy)

## ⚠️ Common Challenges & Solutions

### Time Management

| Challenge                   | Solution                                     |
| --------------------------- | -------------------------------------------- |
| Running over time           | Shorter comments between presentations       |
| Some presentations too long | Gentle signal system (hold up fingers)       |
| Technical delays            | Have a "fun fact" or joke ready while fixing |
| Running under time          | Extend reflection or add more sharing        |

### Presentation Issues

| Challenge          | Solution                                      |
| ------------------ | --------------------------------------------- |
| Student freezes    | "It's okay! Tell us about your favorite joke" |
| Won't speak loudly | Repeat what they say for the class            |
| Code doesn't work  | "Describe what SHOULD happen"                 |
| Jokes fall flat    | YOU laugh and lead applause enthusiastically  |

### Behavior Issues

| Challenge                             | Solution                                           |
| ------------------------------------- | -------------------------------------------------- |
| Students talking during presentations | Quick reminder; strategic seating                  |
| Negative comments                     | Address privately; reinforce kindness expectations |
| One student monopolizing              | Time limits strictly enforced                      |
| Disengagement                         | Direct questions to specific students              |

### Emotional Situations

| Challenge                                    | Solution                                  |
| -------------------------------------------- | ----------------------------------------- |
| Student upset about their project            | Private reassurance; focus on effort      |
| Student disappointed about no award          | "Everyone who finished is a winner!"      |
| Student didn't complete and feels excluded   | Give them an important helper role        |
| Overwhelming emotions (positive or negative) | Acknowledge feelings; move forward gently |
