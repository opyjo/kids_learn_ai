# Term 1, Week 5: Practice & Play! 🎮

## Teacher's Guide

**Course:** Term 1: Hello Python!  
**Age Group:** 9-10 years old  
**Duration:** 60 minutes  
**Term:** 1 of 8 | **Week:** 5 of 8

---

## 📋 Lesson Overview

### Purpose

This is a practice and consolidation lesson—no new concepts are introduced. Students apply everything from Weeks 1-4 through creative challenges. The goal is to build confidence, reinforce skills, and let students experience the joy of creating with code. This lesson also serves as an informal assessment opportunity to identify students who may need additional support before Week 6.

### Learning Objectives

By the end of this lesson, students will have:

1. Practiced all concepts from Weeks 1-4 in creative contexts
2. Completed at least 3 coding challenges
3. Demonstrated ability to combine multiple concepts
4. Experienced creative freedom in programming
5. Self-assessed their understanding through reflection

### Prerequisites

Students should have completed Lessons 1-4 covering:

- Print statements and patterns (Week 1-2)
- Variables: strings and integers (Week 3)
- String methods and concatenation (Week 4)

### Materials Needed

- Computer with internet access (for each student)
- Projector/screen share for demonstrations
- Student handouts with challenge options
- Optional: Printed ASCII art examples
- Optional: Certificates or stickers for participation
- Timer for station rotations (if using stations)

### Pre-Lesson Preparation

1. **Review Week 4 homework** - Note outstanding Mad Libs to celebrate
2. **Decide on structure** - Stations, free choice, or pair programming
3. **Prepare challenge stations** - Print challenge cards if using stations
4. **Set up celebration** - Consider small rewards for participation
5. **Identify struggling students** - Plan extra support during practice time

## 📚 Detailed Teaching Guide

### Part 1: Welcome & Quick Review (5 minutes)

#### Goals

- Build excitement for practice day
- Quick refresh of key concepts
- Set positive, creative tone

#### Script/Talking Points

> "Welcome to Practice & Play day! 🎮 Today is all about FUN. No new concepts—just using everything you've learned to create AWESOME programs!"

**Energy Builder - Speed Round:**

> "Let's warm up with a speed round! I'll ask questions, you shout the answers!"

1. "What command displays text on screen?" → `print()`!
2. "How do we make a line of 20 stars?" → `print("*" * 20)`!
3. "What's a container that stores information?" → Variable!
4. "How do we make text UPPERCASE?" → `.upper()`!
5. "How do we count characters?" → `len()`!

> "Perfect! You remember everything! Now let's CREATE!"

#### Quick Reference Display

Show on screen throughout the lesson:

```python
# Your Toolkit
print("text")           # Display
print("*" * 20)         # Pattern
name = "Alex"           # Variable
name.upper()            # UPPERCASE
name.lower()            # lowercase
len(name)               # Length
```

---

### Part 2: Challenge Instructions (5 minutes)

#### For Station Rotation

> "Today we have 4 coding stations. You'll spend 10 minutes at each:
>
> 🔵 **Station 1:** Profile Creator - Make an 'All About Me' program
> 🟢 **Station 2:** Silly Story Builder - Create a Mad Libs story
> 🟡 **Station 3:** ASCII Art Studio - Make pictures with text
> 🔴 **Station 4:** Greeting Card Factory - Design cards for people
>
> When you hear the timer, save your work and move to the next station!
> Don't worry if you don't finish—just do as much as you can!"

#### For Free Choice

> "Today you get to CHOOSE which challenges to complete! Look at your handout—there are 6 different challenges. You need to complete at least 3, but you can do more if you want!
>
> Pick challenges that sound FUN to you. There's no wrong choice!"

#### For Pair Programming

> "Today you're working with a partner! One person is the DRIVER (they type), and one is the NAVIGATOR (they give directions).
>
> Rules:
>
> 1. Navigator cannot touch the keyboard
> 2. Driver must follow Navigator's directions
> 3. Both must agree on ideas
> 4. We switch roles every 15 minutes!
>
> [Assign partners based on skill levels—pair stronger with weaker for peer teaching]"

---

### Part 3: Challenge Time (35-40 minutes)

#### Teacher Role During Practice

**Circulate constantly and:**

1. **Celebrate effort:** "I love how creative that is!"
2. **Ask questions:** "What does this part do?"
3. **Provide hints:** "Remember how we did patterns with `*`?"
4. **Encourage risk-taking:** "Try something you haven't done before!"
5. **Take notes:** Identify students who struggle with specific concepts

#### Station/Challenge Guidance

**Station 1: Profile Creator**

- Encourage adding MORE variables beyond the template
- Suggest using string methods for name display
- Challenge fast finishers to create a second profile

**Station 2: Silly Story Builder**

- Emphasize SILLINESS—the funnier, the better!
- Remind them about concatenation spacing
- Suggest using `.upper()` for dramatic moments

**Station 3: ASCII Art Studio**

- Start with simple shapes (square, triangle)
- Show them helpful characters: `| - / \ + _ ( ) [ ] { }`
- Encourage planning on paper first if needed

**Station 4: Greeting Card Factory**

- Personal connections matter—make it for real people
- Remind them about decorative borders
- Suggest using `.title()` for names

#### Common Issues and Quick Fixes

| Issue                       | Quick Fix                                          |
| --------------------------- | -------------------------------------------------- |
| "I don't know what to make" | "What's your favorite animal? Let's start there!"  |
| "It's not working"          | "Let's read the error together"                    |
| "I'm done!"                 | "Great! Can you add 3 more features?"              |
| "This is boring"            | "What would make it MORE fun?"                     |
| Frustration                 | "Take a breath. Let's break it into smaller steps" |

#### Timing Alerts (For Stations)

- **8 minutes:** "2 minutes left at this station!"
- **10 minutes:** "Time to rotate! Save your work and move!"

---

### Part 4: Share & Celebrate (5-10 minutes)

#### Goals

- Celebrate student creativity
- Build community through sharing
- Reinforce that there are many ways to solve problems

#### Gallery Walk (Option A)

> "Everyone stand up! Leave your computer showing your FAVORITE creation. We're going to walk around and see what everyone made!"

- Give 3-4 minutes to walk
- Students can leave comments on sticky notes
- Return to seats and share highlights

#### Volunteer Showcase (Option B)

> "Who wants to share their favorite creation? You'll have 30 seconds to show it and tell us what you made!"

- Select 4-5 volunteers
- Celebrate each one: "Round of applause!"
- Point out specific things you like

#### Partner Share (Option C)

> "Turn to someone near you and show them your favorite thing you made. Tell them what you're most proud of!"

- Give 2 minutes for partner sharing
- Ask pairs to share highlights with class

---

### Part 5: Reflection & Homework (5 minutes)

#### Quick Reflection

> "Before we wrap up, take 30 seconds to think:
>
> - What was EASY for you today?
> - What was CHALLENGING?
> - What do you want more practice with?"

**Optional: Quick Poll**

- "Raise your hand if you feel confident with print()" [count]
- "Raise your hand if you feel confident with variables" [count]
- "Raise your hand if string methods make sense" [count]

Use this to identify topics needing Week 6+ review.

#### Introduce Homework

> "For homework, you get to CHOOSE one of three challenges. Pick the one that sounds most fun to you!"

**Briefly explain each option:**

1. **Story Builder** - Write a 10+ line story with variables
2. **ASCII Art Gallery** - Create 3 different text pictures
3. **Word Transformer** - Transform 5 words with all methods

> "All details are in your notes. This is due before our next class!"

#### Preview Week 6

> "Next week is SUPER important. We become BUG DETECTIVES! 🔍 You'll learn how to find and fix mistakes in code—a skill EVERY professional programmer needs!"

## 🎯 Assessment & Differentiation

### Informal Assessment Checklist

Use this lesson to assess student progress. Note observations for each student:

| Skill                          | Observed? | Notes |
| ------------------------------ | --------- | ----- |
| Uses print() correctly         | ☐         |       |
| Creates string patterns        | ☐         |       |
| Creates variables              | ☐         |       |
| Distinguishes strings/integers | ☐         |       |
| Uses string methods            | ☐         |       |
| Concatenates properly          | ☐         |       |
| Combines multiple concepts     | ☐         |       |
| Works independently            | ☐         |       |
| Shows creativity               | ☐         |       |

### Identifying Struggling Students

Watch for these warning signs:

- Copies examples exactly without modification
- Frozen/not typing for extended periods
- Same errors repeated multiple times
- Frustration or giving up quickly
- Doesn't ask for help when stuck

**Intervention:** Pair with stronger student, provide simpler template, schedule 1:1 time before Week 6

### Identifying Advanced Students

Signs of readiness for extension:

- Completes challenges quickly and correctly
- Asks "what else can I do?"
- Helps peers naturally
- Experiments beyond requirements
- Combines concepts in creative ways

**Extension Ideas:**

- Introduce comments: `# This is a comment`
- Show `.replace()` method
- Challenge: Create an interactive menu
- Preview: What's coming in Term 2

### Homework Assessment: "Choose Your Challenge"

| Criteria           | Full Credit           | Partial Credit    | Needs Work           |
| ------------------ | --------------------- | ----------------- | -------------------- |
| Meets requirements | All requirements met  | Most requirements | Missing key elements |
| Week 1-4 concepts  | Uses 3+ concepts      | Uses 2 concepts   | Uses 1 concept       |
| Creativity         | Original and engaging | Some creativity   | Minimal effort       |
| Code quality       | Runs perfectly        | Minor errors      | Major errors         |
| Organization       | Well-formatted        | Readable          | Messy                |

### Differentiation Strategies

#### For Students Who Need More Support

- Provide completed example to modify (not create from scratch)
- Pair with peer mentor during challenges
- Reduce requirements (5 lines instead of 10, 2 art pieces instead of 3)
- Focus on just one challenge done well
- Offer "office hours" before Week 6

#### For Advanced Students

- Challenge: Combine TWO challenges into one program
- Introduce string slicing: `name[0]` for first letter
- Have them create a "tutorial" for a concept
- Ask them to find bugs in intentionally broken code
- Preview loops: "What if we could repeat things automatically?"

## ⚠️ Common Pitfalls & Solutions

### Engagement Issues

| Problem                    | Solution                                                |
| -------------------------- | ------------------------------------------------------- |
| Student is bored           | "What would make this more challenging for you?"        |
| Student won't start        | "Let's just get ONE print statement working"            |
| Student only wants to play | "Show me ONE completed challenge, then you can explore" |
| Pairs not working well     | Restructure pairs or switch to individual work          |

### Technical Issues

| Problem                  | Solution                                  |
| ------------------------ | ----------------------------------------- |
| Lost work                | Teach Ctrl+Z to undo; remind about saving |
| Same error over and over | Stop and explain; demonstrate correct way |
| Trinket running slowly   | Refresh the page; reduce code complexity  |
| Can't find challenge     | Share screen showing where challenges are |

### Time Management

| Problem                     | Solution                                    |
| --------------------------- | ------------------------------------------- |
| Stations taking too long    | Reduce to 8 minutes per station             |
| Not enough time for sharing | Shorten to partner share only               |
| Running out of time         | Skip reflection, assign as written homework |
| Students finish too fast    | Have bonus challenges ready                 |
