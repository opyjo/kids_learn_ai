## 📋 Lesson Overview

### Purpose

This is the culminating celebration of Term 1. The primary goals are to: (1) give every student a moment to shine by presenting their Mad Libs game, (2) celebrate individual and collective achievements, (3) build confidence through public presentation, and (4) generate excitement for Term 2: Math Wizard. This lesson is about celebration and closure, not new content.

### Learning Objectives

By the end of this lesson, students will have:

1. Presented their completed Mad Libs game to peers
2. Practiced public speaking and presentation skills
3. Celebrated their own and classmates' achievements
4. Reflected on their Term 1 learning journey
5. Built excitement for Term 2: Math Wizard

### Prerequisites

- Students should have completed their Mad Libs game project (Week 8)
- All projects should be submitted/accessible before class
- Any students who didn't complete should have a plan (see Differentiation)

### Materials Needed

- Video conferencing platform (Zoom, Google Meet, etc.)
- Screen sharing enabled for all students
- Digital certificates (emailed or shared via chat)
- Awards/recognition materials (digital badges or slides)
- Timer for presentations
- Chat window ready for audience participation
- Screenshot tool for class photo (optional)
- Backup plan for technical issues (student can share code link in chat if screen share fails)

### Pre-Lesson Preparation

1. **Test all student projects** — Verify Trinket links work before class
2. **Prepare presentation order** — Mix skill levels, volunteers first
3. **Prepare digital certificates** — Personalized for each student (PDF or image to email/share)
4. **Prepare awards** — Decide on categories and recipients
5. **Test screen sharing** — Ensure students know how to share their screen
6. **Prepare Term 2 preview** — Exciting snippets about Math Wizard to share
7. **Have student Trinket links ready** — Backup in case a student can't screen share

## 📚 Detailed Teaching Guide

### Part 1: Welcome & Energy Setting (2 minutes)

#### Goals

- Create exciting, celebratory atmosphere
- Set positive tone for presentations
- Reduce nervousness through collective excitement

#### Script/Talking Points

> "Welcome to the GAME SHOWCASE! 🎉
>
> This is it — the moment you've been working toward! Today is YOUR day to shine!
>
> You've learned SO much this term. You've built REAL games. And now you get to show them off!
>
> A few things before we start:
>
> - Everyone here has accomplished something amazing
> - We're all here to support each other
> - When someone asks for a silly word — shout out the silliest thing you can think of!
> - After each presentation, we clap and cheer!
>
> Are you ready to celebrate?!"

**Get an enthusiastic response!**

---

### Part 2: Final Preparations (8 minutes)

#### Goals

- Give students last chance to fix issues
- Reduce anxiety through preparation
- Handle any technical problems

#### Setup

> "You have 8 minutes to:
>
> 1. Test your Mad Libs game ONE more time
> 2. Make sure you know how to share your screen
> 3. Practice what you'll say
> 4. Take some deep breaths!
>
> If you have any last-second issues, send me a private message in the chat."

#### Teacher Actions During Prep

1. **Check in via chat** — Ask each student to confirm they're ready with a thumbs up emoji
2. **Identify issues** — Note any projects that won't run or students with tech problems
3. **Quick fixes only** — No major debugging now; help with screen share setup if needed
4. **Reassure nervous students** — Private message: "You've got this!"
5. **Confirm presentation order** — Finalize the list and share it in the chat

#### Handling Non-Complete Projects

If a student didn't finish:

- Have them present what they DO have
- Focus on what they learned, not what's missing
- Consider pairing with a complete project to "co-present"
- NEVER shame or exclude — everyone presents something

#### 2-Minute Warning

> "Two more minutes! Make sure your game is ready to run. Remember, you'll introduce yourself, run your code, and share one thing you're proud of. That's it! Easy!"

---

### Part 3: Showcase Presentations (30 minutes)

#### Goals

- Every student presents successfully
- Maintain supportive, celebratory atmosphere
- Keep presentations moving on time

#### Presentation Structure

Each presentation should follow this format (~2–3 minutes):

1. **Introduction (30 sec)**
   - Student name
   - Game theme (if they chose one)
2. **Screen Share & Run the Game (1–1.5 min)**
   - Student shares their screen
   - Runs the code for everyone to see
   - Ask audience to type words in the chat for the blanks
   - Read the final silly story aloud
3. **Reflection (30 sec)**
   - One thing they're proud of
   - OR one thing they learned
4. **Applause (15 sec)**
   - Lead enthusiastic applause (everyone unmutes!)
   - Positive comment from teacher
   - Student stops screen share

#### Virtual Mad Libs Showcase Tips

The Mad Libs format is naturally interactive — use this to your advantage in a virtual setting:

- **Use the chat for word suggestions** — Have the class type silly words in the chat for the blanks
- **Pick words from the chat** — The presenter (or teacher) picks from the flood of chat suggestions
- **Read stories dramatically** — Ham it up when reading the final story
- **Celebrate the silliness** — The funnier the result, the better!
- **Emoji reactions** — Encourage students to use emoji reactions during presentations

#### Time Management

With 15–20 students, you have ~2 minutes each.

| Class Size | Time Per Student | Strategy                 |
| ---------- | ---------------- | ------------------------ |
| 10–12      | 2.5–3 min        | Full presentations       |
| 13–18      | 2 min            | Streamlined format       |
| 19–25      | 1.5 min          | Quick showcase style     |
| 25+        | Consider groups   | Split into two sessions  |

**Use a timer!** Gently signal when time is running low.

#### Leading Applause

After EVERY presentation:

- Start clapping immediately
- Make eye contact with the presenter
- Say something specific and positive:
  - "Great job, [name]! That story was hilarious!"
  - "I loved how you used [specific detail] in your game!"
  - "Awesome work! Your [theme/creativity] really stood out!"

#### Handling Technical Issues (Virtual)

| Issue                    | Quick Solution                                                  |
| ------------------------ | --------------------------------------------------------------- |
| Can't share screen       | Have student paste their Trinket link in chat; teacher shares   |
| Project won't load       | Have student describe what it does while you troubleshoot       |
| Trinket is slow          | Fill time with questions about the project                      |
| Code has bug             | "This is a great debugging opportunity!"                        |
| Student freezes          | Gently prompt: "Tell us about your Mad Libs theme"              |
| Audio/mic issues         | Student types in chat; teacher reads aloud for the class        |
| Internet lag             | Be patient; ask student to turn off camera to save bandwidth    |

#### Presentation Order Strategies

**Option 1: Volunteers First** (Recommended)

- Ask for volunteers initially
- Nervous students see examples first
- Builds momentum

**Option 2: Strategic Order**

- Confident student first (sets good example)
- Mix skill levels throughout
- Save a strong one for the end

## 🎯 Assessment & Differentiation

### Assessment Focus

This lesson is about **celebration, not evaluation**. However, note:

**Observation Points:**

- Presentation confidence level
- Ability to explain their project
- Peer support behaviors
- Engagement during others' presentations

**Use for Term 2 Planning:**

- Which students need more support?
- Which are ready for challenges?
- Which topics need review at the start of Term 2?

### Differentiation Strategies

#### For Students Who Didn't Complete

**NEVER exclude a student from presenting.**

Options:

1. Present what they have (even if incomplete)
2. Describe what they TRIED to build
3. Share what they learned from the attempt
4. "Co-present" with a partner who completed

**Script for incomplete project:**

> "Even though [name]'s project isn't quite finished, they learned SO much about [concepts]. Tell us about what you worked on..."

#### For Nervous Presenters

- Offer a private practice run before class starts
- Allow camera-off presentation if being on camera is too scary
- Reduce requirements (just share screen and run code, no explanation needed)
- Present earlier in the order (less buildup anxiety)
- Teacher can share the student's screen for them if screen sharing is too stressful

#### For Overconfident Students

- Channel energy positively
- Give them a "helper" role for nervous students
- Ensure they're respectful during others' presentations
- Consider last spot (big finale energy)

## ⚠️ Common Challenges & Solutions

### Time Management

| Challenge                   | Solution                                     |
| --------------------------- | -------------------------------------------- |
| Running over time           | Shorter comments between presentations       |
| Some presentations too long | Gentle signal system (hold up fingers)        |
| Technical delays            | Have a "fun fact" or joke ready while fixing  |
| Running under time          | Extend reflection or add more sharing         |

### Presentation Issues

| Challenge          | Solution                                               |
| ------------------ | ------------------------------------------------------ |
| Student freezes    | "It's okay! Tell us about your Mad Libs theme"         |
| Won't speak loudly | Repeat what they say for the class                     |
| Code doesn't work  | "Describe what SHOULD happen"                          |
| Story falls flat   | YOU laugh and lead applause enthusiastically            |

### Behavior Issues

| Challenge                             | Solution                                           |
| ------------------------------------- | -------------------------------------------------- |
| Students chatting off-topic            | Remind about chat expectations; use mute all       |
| Negative comments                     | Address privately; reinforce kindness expectations |
| One student monopolizing              | Time limits strictly enforced                      |
| Disengagement                         | Direct questions to specific students              |
