# Term 4, Lesson 7: Project - Big Adventure! 🎮

## Teacher's Guide

**Course:** Term 4: More Choices  
**Age Group:** 9-10 years old  
**Duration:** 60 minutes  
**Term:** 4 of 8 | **Week:** 7 of 8

---

## 📋 Lesson Overview

### Purpose

This is the term project lesson where students combine ALL their skills (if, else, elif, and, or, menus, quizzes) into one complete expanded adventure game. This is a synthesis and application lesson that demonstrates mastery of all concepts learned this term. Students will build something bigger and more complex than their Term 3 project.

### Learning Objectives

By the end of this lesson, students will have:

1. Built a complete expanded adventure game
2. Used elif for multiple choices (not just yes/no)
3. Created at least 4 different endings
4. Added a scoring or inventory system
5. Combined all decision-making skills in one project

### Materials Needed

- Computer with internet access (for each student)
- Projector/screen share for demonstrations
- Trinket accounts
- Whiteboard for showing structure
- Optional: Sample adventure games to inspire
- Optional: Planning worksheets

### Pre-Lesson Preparation

1. **Review all concepts** — if, else, elif, and, or, menus
2. **Prepare example structure** — Have a working example ready
3. **Think about themes** — Space, fantasy, mystery, etc.
4. **Prepare common issues** — Wrong elif usage, missing endings
5. **Plan support strategy** — How to help students who get stuck

## 📚 Detailed Teaching Guide

### Part 1: Introduction (5 minutes)

#### Goals

- Set context for the project
- Review requirements clearly
- Get students excited
- Show what makes this different from Term 3

#### The Introduction

> "This is it—your TERM PROJECT! You've learned SO MUCH this term, and now you're going to put it ALL together to build an amazing expanded adventure game!"

**Show the differences from Term 3:**

| Term 3 | Term 4 |
|--------|--------|
| 2-3 endings | 4+ endings |
| Yes/no choices | Multiple choices (elif) |
| Basic if/else | if/elif/else, and/or |
| Simple story | Longer, more complex |

> "This project will be BIGGER and BETTER! You'll use elif for multiple choices, create more endings, and add scoring or inventory!"

**Review requirements:**
- Welcome message with borders
- Longer story with more branches
- elif for multiple choices (1, 2, 3, etc.)
- At least 4 different endings
- Scoring or inventory system
- No bugs!

#### Teaching Tips

- Build excitement
- Emphasize this is their chance to show off
- Make requirements clear
- Show examples if helpful
- Keep it brief—get them building

---

### Part 2: Planning (10 minutes)

#### Goals

- Students plan their adventure
- Think about structure before coding
- Avoid getting stuck later

#### Planning Activity

> "Before we code, let's plan! Think about:"

**Guide them through planning:**

1. **Theme:** What's your adventure about?
2. **Beginning:** How does it start?
3. **First choice:** What are the options? (Use elif - 3+ options)
4. **Paths:** What happens on each path?
5. **More choices:** What choices come next?
6. **Endings:** What are the 4+ endings?
7. **Scoring/Inventory:** How will you track progress?

**Give them 5-7 minutes to plan:**
- Can sketch on paper
- Can write notes
- Can discuss with neighbors

**Circulate and help:**
- Help brainstorm themes
- Guide structure
- Ensure they understand elif requirement
- Make sure they plan 4+ endings

#### Teaching Tips

- Don't skip planning—it helps!
- Give concrete time limit
- Help students who are stuck
- Encourage creativity
- Check that they understand requirements

---

### Part 3: Building (35 minutes)

#### Goals

- Students build their adventure
- Apply all skills learned
- Create something they're proud of

#### Building Time

> "Now it's time to build! Start with the welcome screen, then the story beginning, then your first choice with elif. Build step by step, test as you go!"

**Provide structure reminders:**

1. Start with welcome screen
2. Add story beginning
3. First choice with elif (3+ options)
4. Continue each path
5. Add more choices
6. Create 4+ endings
7. Add scoring/inventory
8. Test everything!

**Circulate and support:**
- Help with structure
- Help with elif syntax
- Help with scoring/inventory
- Fix bugs immediately
- Encourage creativity
- Celebrate progress

**Common issues to watch for:**
- Using yes/no instead of elif
- Not enough endings
- Missing scoring/inventory
- Syntax errors
- Logic errors

#### Teaching Tips

- Give them space to create
- Help when stuck, but don't hover
- Fix bugs as they appear
- Encourage testing
- Celebrate small wins
- Keep energy positive

---

### Part 4: Testing & Polish (7 minutes)

#### Goals

- Students test their adventures
- Fix any bugs
- Polish the presentation

#### Testing Time

> "Now let's test! Try all your choices, walk through all paths, check all endings. Fix any bugs you find!"

**Testing checklist:**
- [ ] All choices work
- [ ] All paths work
- [ ] All endings are different
- [ ] Scoring/inventory works
- [ ] No errors
- [ ] Looks good

**Help students:**
- Test with them
- Find bugs
- Fix issues
- Polish presentation

#### Teaching Tips

- Emphasize testing is important
- Help find bugs
- Fix issues quickly
- Encourage polish

---

### Part 5: Wrap-up (3 minutes)

#### Goals

- Preview next week
- Build excitement for showcase
- Remind about homework

#### The Wrap-up

> "What amazing adventures you've built! Next week, we're going to SHOWCASE them! You'll present your adventure, play each other's games, and earn your 'Decision Master' badge!"

**Homework reminder:**
> "Finish polishing your adventure for next week's showcase! Make sure it's perfect!"

**Build excitement:**
> "I can't wait to see everyone's adventures next week! You've built something amazing!"

## 🎯 Assessment & Differentiation

### Formative Assessment

**During class, observe:**
- [ ] Does student use elif correctly?
- [ ] Does student have 4+ endings?
- [ ] Does student have scoring/inventory?
- [ ] Is the adventure complete?
- [ ] Does it run without errors?

### Project Assessment: "Big Adventure"

| Criteria | Exceeds (3) | Meets (2) | Developing (1) |
|----------|-------------|-----------|----------------|
| **elif usage** | Uses elif correctly throughout | Uses elif correctly | Struggles with elif |
| **Number of endings** | 6+ endings | 4 endings | Fewer than 4 |
| **Story complexity** | Long, complex story | Good story | Basic story |
| **Scoring/inventory** | Well-implemented | Implemented | Missing or basic |
| **Choices** | 5+ choices with elif | 3+ choices with elif | Basic choices |
| **Code runs** | No errors | Minor issues | Major errors |
| **Creativity** | Highly creative | Creative | Basic |

### Differentiation Strategies

#### For Students Who Need More Support

- Provide a template with structure
- Focus on getting 4 endings first
- Simplify scoring/inventory
- Pair with a peer
- Provide step-by-step guidance
- Reduce complexity if needed

#### For Advanced Students

- Challenge with 6+ endings
- Encourage complex nested choices
- Challenge with advanced scoring
- Ask them to help peers
- Encourage extra features
- Challenge with longer story

## ⚠️ Common Pitfalls & Solutions

### Technical Issues

| Problem | Prevention | Solution |
|---------|------------|----------|
| Using yes/no instead of elif | Emphasize elif requirement | "Use elif with 1, 2, 3, etc." |
| Not enough endings | Emphasize 4+ requirement | "Plan your endings first" |
| Missing scoring/inventory | Emphasize requirement | "Add score = 0 at start" |
| Syntax errors | Review elif syntax | "elif needs colon and indentation" |

### Conceptual Issues

| Problem | Signs | Solution |
|---------|-------|----------|
| Doesn't understand elif | Uses if/else instead | "elif is for 3+ options" |
| Struggles with structure | Gets lost | "Plan first, then code" |
| Overwhelmed | Stuck, not building | "Start simple, add complexity" |
