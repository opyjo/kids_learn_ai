## 📋 Lesson Overview

### Purpose

This is the term project week. Students will create an ASCII Art Generator that combines all the skills they've learned: loops, patterns, decisions, and user input. This is a comprehensive project that demonstrates mastery of Term 6 concepts.

### Learning Objectives

By the end of this lesson, students will be able to:

1. Combine loops, patterns, and decisions in one program
2. Create an interactive menu system
3. Use input() and if/elif/else together
4. Build a complete, working program
5. Prepare for next week's showcase

### Materials Needed

- Computer with internet access (for each student)
- Projector/screen share for demonstrations
- Trinket accounts
- Whiteboard for project planning
- Optional: Example projects to show

### Pre-Lesson Preparation

1. **Review project requirements** — Know what's required vs bonus
2. **Prepare example** — Have a working ASCII Art Generator ready
3. **Plan time management** — Structure the 60 minutes effectively
4. **Prepare troubleshooting** — Know common issues students will face
5. **Think about differentiation** — How to support all students

## 📚 Detailed Teaching Guide

### Part 1: Project Introduction (10 minutes)

#### Goals

- Explain project requirements clearly
- Show an example project
- Build excitement
- Answer questions

#### Script/Talking Points

> "Welcome to PROJECT WEEK! 🎉 This is your chance to show everything you've learned this term. You're going to create an ASCII Art Generator—a program that lets users choose patterns and creates beautiful art!"

**Show Example Project:**

Run through a complete example:
1. Show welcome screen
2. Show menu
3. Choose a pattern
4. Show pattern output
5. Show ending message

> "This is what you're building! But make it YOURS—add your name, your favorite patterns, your creative touches!"

**Explain Requirements:**

Go through the checklist:
- Welcome screen ✓
- Menu with 3+ patterns ✓
- User input ✓
- Pattern creation ✓
- Ending message ✓
- Comments ✓
- No bugs ✓

**Explain Bonus Features:**

- More patterns
- Customization
- Enhanced features
- Error handling

#### Teaching Tips

- Be enthusiastic—this is exciting!
- Show a complete example to inspire
- Make requirements clear
- Encourage creativity

---

### Part 2: Planning Time (5 minutes)

#### Goals

- Students plan their patterns
- Think about structure
- Prepare to build

#### Activity

> "Before you start coding, let's plan! Think about:"
- What patterns do you want to include?
- What will your welcome message say?
- How will you make it special?

Give students 5 minutes to plan. Walk around and help with ideas.

#### Teaching Tips

- Encourage planning before coding
- Help students choose appropriate patterns
- Suggest creative touches
- Keep it brief—they're eager to code!

---

### Part 3: Building Time (35 minutes)

#### Goals

- Students create their projects
- Provide support as needed
- Encourage independence
- Celebrate progress

#### Structure

**First 10 minutes:**
- Students start building
- Help with setup (welcome screen, menu)
- Check in with each student

**Middle 15 minutes:**
- Students build patterns
- Circulate and help with loops
- Fix bugs as they arise
- Encourage creativity

**Last 10 minutes:**
- Students finish patterns
- Add ending messages
- Add comments
- Test their code

#### Teacher Circulation

During this time:
- **Check progress** — Are they on track?
- **Help with bugs** — Fix errors quickly
- **Encourage** — Celebrate what they're creating
- **Differentiate** — Support struggling students, challenge advanced ones
- **Answer questions** — Be available and helpful

#### Common Issues to Watch For

1. **Menu input** — Make sure they compare strings: `choice == "1"`
2. **Loop syntax** — Check indentation, colons
3. **Pattern logic** — Help with nested loops
4. **Testing** — Encourage frequent testing

#### Teaching Tips

- Stay mobile—circulate constantly
- Don't give answers—guide to solutions
- Celebrate small wins
- Help students help each other
- Keep energy high

---

### Part 4: Testing & Polish (7 minutes)

#### Goals

- Students test their complete programs
- Fix any remaining bugs
- Add final touches
- Prepare for showcase

#### Activity

> "Time to test! Run through your complete program:"
- Does the welcome screen show?
- Does the menu work?
- Do all patterns work?
- Does the ending message show?
- Are there any bugs?

Give students time to test and fix issues. Help with any remaining problems.

#### Teaching Tips

- Encourage thorough testing
- Help fix final bugs
- Celebrate completed projects
- Remind about comments

---

### Part 5: Wrap-up & Preview (3 minutes)

#### Goals

- Celebrate what they've accomplished
- Preview next week's showcase
- Set expectations for homework

#### Script

> "WOW! Look what you've created! You've built complete programs that combine loops, patterns, and decisions. That's AMAZING!"

**Preview Next Week:**

> "Next week is the SHOWCASE! You'll present your ASCII Art Generator, show off your patterns, and see what your classmates created. Plus, you'll earn your 'Loop Master' badge!"

**Homework Reminder:**

> "For homework, finish polishing your project. Make sure it's perfect and ready to showcase!"

#### Teaching Tips

- Celebrate their achievements
- Build excitement for showcase
- Set clear expectations

## 🤖 AI Activity: AI Art Discussion (5-10 minutes)

### What to Do

1. **Introduction** (1 min)
   > "You just created art with code - ASCII art! But here's a big question: What about AI art? Is it 'real' art?"

2. **Art Discussion** (4-5 min)
   - Ask: "What is art? Can AI be creative?"
   - Show examples: "AI can create images, music, stories - is that art?"
   - Discuss: "What's the difference between human art and AI art?"
   - Brainstorm: "Can you tell the difference? Should AI art be treated the same as human art?"

3. **Discussion** (2-3 min)
   - "Is AI art 'real' art?"
   - "Can AI be creative?"
   - "What's the difference between your ASCII art and AI art?"

4. **Connection** (1 min)
   > "You're creating art with code, just like AI does! But these are important questions people are discussing right now: Is AI art real art? Can AI be creative? You're part of this conversation!"

### Discussion Questions

- "Is AI art 'real' art?"
- "Can AI be creative?"
- "What's the difference between human art and AI art?"

### Teaching Tips

- Encourage all opinions - there's no "right" answer
- Keep it age-appropriate - focus on creativity and expression
- Use examples: show AI-generated images if appropriate
- Connect to their project: "You're creating art with code - just like AI!"
- Emphasize thinking critically: "These are questions people are asking right now!"

### Alternative Activity (If Short on Time)

- Quick discussion: "Who thinks AI can create art?"
- Connect: "You're creating art with code - just like AI!"
- Emphasize: "These are important questions people are discussing!"

## 🎯 Assessment & Differentiation

### Formative Assessment

**During class, observe:**

- [ ] Can student create welcome screen?
- [ ] Can student create menu?
- [ ] Can student use input() correctly?
- [ ] Can student create patterns with loops?
- [ ] Can student combine if/elif/else?
- [ ] Is project complete and working?

### Project Assessment: "ASCII Art Generator"

| Criteria              | Exceeds (3)                    | Meets (2)                  | Developing (1)           |
| --------------------- | ------------------------------ | -------------------------- | ------------------------ |
| **Welcome screen**    | Creative, well-formatted       | Basic welcome              | Missing or incomplete    |
| **Menu**              | Clear, 4+ options, well-designed | 3 options, clear          | Less than 3, unclear     |
| **User input**        | Works perfectly, error handling | Works correctly            | Doesn't work             |
| **Patterns**          | 4+ patterns, creative, complex | 3 patterns, correct        | Less than 3, incorrect  |
| **Code quality**      | Well-commented, organized      | Some comments, organized  | No comments, messy       |
| **Runs without errors** | Perfect, bonus features       | Runs correctly             | Has errors               |
| **Creativity**         | Very creative, unique          | Some creativity             | Basic only               |

### Differentiation Strategies

#### For Students Who Need More Support

- Provide starter template
- Focus on 3 basic patterns (square, triangle, simple)
- Pair with peer helper
- Break into smaller steps
- Provide pattern code examples
- Reduce to 2 patterns if needed

#### For Advanced Students

- Challenge with 5+ patterns
- Add customization features
- Create conditional patterns
- Add error handling
- Help other students
- Create multiple pattern sizes

## ⚠️ Common Pitfalls & Solutions

### Technical Issues

| Problem                  | Prevention                    | Solution                                    |
| ------------------------ | ----------------------------- | ------------------------------------------- |
| Input comparison         | Emphasize string comparison   | "Use choice == '1' not choice == 1"         |
| Loop syntax errors       | Review loops before project   | Check indentation, colons                   |
| Pattern not showing      | Test each pattern separately  | Check loops, end="", print()                |
| Menu not working         | Test input handling           | Check if/elif/else logic                    |

### Time Management Issues

| Problem                  | Signs                          | Solution                                    |
| ------------------------ | ------------------------------ | ------------------------------------------- |
| Running out of time      | Not finished by end of class   | Prioritize: 3 patterns first, then extras   |
| Perfectionism            | Stuck on one pattern           | "Get it working, then improve"             |
| Overwhelmed              | Too many features              | "Start simple, add if time allows"          |
