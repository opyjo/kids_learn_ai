# Term 7, Lesson 7: Project - Guessing Game 🎮

## Teacher's Guide

**Course:** Term 7: Game Builder  
**Age Group:** 9-10 years old  
**Duration:** 60 minutes  
**Term:** 7 of 8 | **Week:** 7 of 8

## 📋 Lesson Overview

### Purpose

This is the term project week. Students will create a complete Number Guessing Game that combines all skills learned: game loops, random numbers, score tracking, and decisions. This is a comprehensive project that demonstrates mastery of Term 7 concepts.

### Learning Objectives

By the end of this lesson, students will be able to:

1. Combine all features into one complete game
2. Create a playable number guessing game
3. Implement higher/lower hints
4. Track score and attempts
5. Build a complete, working program

### Materials Needed

- Computer with internet access (for each student)
- Projector/screen share for demonstrations
- Trinket accounts
- Whiteboard for project planning
- Optional: Example projects to show

### Pre-Lesson Preparation

1. **Review project requirements** — Know what's required vs bonus
2. **Prepare example** — Have a working guessing game ready
3. **Plan time management** — Structure the 60 minutes effectively
4. **Prepare troubleshooting** — Know common issues students will face
5. **Think about differentiation** — How to support all students

## 📚 Detailed Teaching Guide

### Part 1: Project Introduction (10 minutes)

#### Goals

- Explain project requirements clearly
- Show an example project
- Build excitement
- Answer questions

#### Script/Talking Points

> "Welcome to PROJECT WEEK! 🎉 This is your chance to show everything you've learned. You're going to create a Number Guessing Game—a complete, playable game!"

**Show Example Project:**

Run through a complete example:
1. Show welcome screen
2. Generate random number
3. Show guessing with hints
4. Show score tracking
5. Show "Play again?"
6. Show final statistics

> "This is what you're building! But make it YOURS!"

**Explain Requirements:**

Go through the checklist:
- Welcome screen ✓
- Random number ✓
- Guessing with hints ✓
- Score tracking ✓
- Game loop ✓
- Final stats ✓

#### Teaching Tips

- Be enthusiastic
- Show complete example
- Make requirements clear
- Encourage creativity

---

### Part 2: Planning Time (5 minutes)

#### Goals

- Students plan their games
- Think about features
- Prepare to build

#### Activity

> "Before coding, let's plan! Think about:"
- What will your welcome message say?
- What range will you use? (1-20? 1-100?)
- How will you make it special?

Give students 5 minutes to plan. Walk around and help.

#### Teaching Tips

- Encourage planning
- Help with ideas
- Keep it brief

---

### Part 3: Building Time (35 minutes)

#### Goals

- Students create their projects
- Provide support
- Encourage independence

#### Structure

**First 10 minutes:**
- Students start building
- Help with setup
- Check in with each student

**Middle 15 minutes:**
- Students build core features
- Circulate and help
- Fix bugs as they arise

**Last 10 minutes:**
- Students finish features
- Add enhancements
- Test their code

#### Teacher Circulation

- Check progress
- Help with bugs
- Encourage
- Differentiate
- Answer questions

#### Common Issues to Watch For

1. **Nested loops** — Guessing loop inside game loop
2. **Hint logic** — if/elif/else for higher/lower
3. **Score updates** — Update inside loop
4. **Variable scope** — Initialize before loops

#### Teaching Tips

- Stay mobile
- Don't give answers—guide
- Celebrate small wins
- Help students help each other

---

### Part 4: Testing & Polish (7 minutes)

#### Goals

- Students test complete games
- Fix remaining bugs
- Add final touches

#### Activity

> "Time to test! Run through your complete game:"
- Does welcome screen show?
- Does random number work?
- Do hints work?
- Does score track?
- Does "Play again?" work?
- Are final stats shown?

Give students time to test and fix. Help with any remaining problems.

#### Teaching Tips

- Encourage thorough testing
- Help fix final bugs
- Celebrate completed projects

---

### Part 5: Wrap-up & Preview (3 minutes)

#### Goals

- Celebrate achievements
- Preview showcase
- Set expectations

#### Script

> "WOW! Look what you've created! Complete, playable games!"

**Preview Next Week:**

> "Next week is the SHOWCASE! You'll present your game, play classmates' games, and earn your 'Game Developer' badge!"

**Homework Reminder:**

> "For homework, finish polishing your game. Make sure it's perfect!"

#### Teaching Tips

- Celebrate achievements
- Build excitement for showcase
- Set clear expectations

## 🎯 Assessment & Differentiation

### Formative Assessment

**During class, observe:**

- [ ] Can student create game loop?
- [ ] Can student generate random numbers?
- [ ] Can student implement hints?
- [ ] Can student track score?
- [ ] Is project complete and working?

### Project Assessment: "Number Guessing Game"

| Criteria              | Exceeds (3)                    | Meets (2)                  | Developing (1)           |
| --------------------- | ------------------------------ | -------------------------- | ------------------------ |
| **Welcome screen**    | Creative, well-formatted       | Basic welcome              | Missing or incomplete    |
| **Random number**      | Works, maybe difficulty levels | Works correctly            | Doesn't work             |
| **Guessing with hints** | Works perfectly, creative     | Works correctly            | Doesn't work             |
| **Score tracking**     | Multiple types, creative       | Basic tracking             | Missing or incorrect     |
| **Game loop**          | Works perfectly, enhanced      | Works correctly            | Doesn't work             |
| **Code quality**       | Well-commented, organized      | Some comments              | No comments, messy       |
| **Runs without errors** | Perfect, bonus features       | Runs correctly             | Has errors               |

### Differentiation Strategies

#### For Students Who Need More Support

- Provide starter template
- Focus on basic features first
- Pair with peer helper
- Break into smaller steps
- Reduce to essential features

#### For Advanced Students

- Challenge with difficulty levels
- Add bonus features
- Help other students
- Create multiple game modes

## ⚠️ Common Pitfalls & Solutions

### Technical Issues

| Problem                  | Prevention                    | Solution                                    |
| ------------------------ | ----------------------------- | ------------------------------------------- |
| Nested loop confusion    | Show structure clearly        | "Outer loop = game, inner loop = guessing"  |
| Hint logic errors        | Practice if/elif/else         | "Check: <, ==, >"                          |
| Score not updating       | Show where to update          | "Update inside loop after win"             |
| Variable scope          | Initialize before loops        | "Set variables before while"                |

### Time Management Issues

| Problem                  | Signs                          | Solution                                    |
| ------------------------ | ------------------------------ | ------------------------------------------- |
| Running out of time      | Not finished                   | Prioritize: core features first             |
| Perfectionism            | Stuck on one feature           | "Get it working, then improve"              |
