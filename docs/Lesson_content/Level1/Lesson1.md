### Section 1: Welcome & Hook (5 minutes)

**Goal:** Create excitement and establish relevance immediately.

**Script/Talking Points:**

### Section 2: What is Python? (8 minutes)

**Goal:** Define Python in kid-friendly terms.

**Key Points to Cover:**

1. Python is a language that lets us talk to computers
2. Named after Monty Python (comedy show), not the snake
3. Created in 1991 by Guido van Rossum

**Analogy to Use:**

> "Think about how you give directions to get to your house. You might say: 'Go straight, turn left at the big tree, it's the blue house on the right.' You have to be really specific, right? Python is like that - we give computers very specific instructions, and they follow them exactly."

**Check for Understanding:**

- "If I said 'go to the place' - would that be good directions? Why not?"
- "What makes good directions?"

**Visual:** If possible, show Python logo and a simple code snippet on screen.

---

### Section 3: Robot Commander Game (7 minutes)

**Goal:** Demonstrate the importance of precise instructions through physical activity.

**Setup:**

1. Explain that you'll be the "robot" first
2. Students give you instructions to complete a simple task

**Activity Instructions:**

**Round 1 - Instructor is the Robot (3 min):**

> "I'm going to be a robot, and YOU have to tell me how to pick up this pen from my desk. But here's the rule - I only understand very specific commands. Ready?"

**Commands the robot understands:**

- "Move hand forward [number] inches"
- "Move hand left/right [number] inches"
- "Open hand"
- "Close hand"
- "Move hand down [number] inches"
- "Move hand up [number] inches"

**Instructor behavior:** Act literally. If they say "grab the pen" - look confused. "What does 'grab' mean? I need step-by-step instructions!"

**Debrief Questions:**

- "What was hard about giving instructions to the robot?"
- "What happened when the instructions weren't specific enough?"
- "This is EXACTLY what coding is like! Computers need very specific instructions."

**Round 2 - Student Pairs (4 min):**
If time permits and class size allows, have students in breakout rooms give each other instructions for a simple task (like touching their nose with eyes closed).

**Instructor Note:** This activity is crucial for building the mental model of how coding works. Don't skip it even if running short on time - cut from Section 4 instead.

### Section 5: Your First Code - Live Coding (15 minutes)

**Goal:** Students successfully write and run their first Python program.

**Setup Checkpoint (2 min):**

- Ensure all students have coding environment open
- Share your screen
- Have students give thumbs up when ready

**Demo 1: Hello World (3 min)**

```python
print("Hello, World!")
```

**Talk through as you type:**

> "Watch carefully. I type p-r-i-n-t, then open parenthesis, then quotation marks, then my message, then closing quotation marks, then closing parenthesis. Let's run it!"

**Student Practice (2 min):**

- Students type exactly what you typed
- Walk through anyone having issues
- Celebrate when it works! "You just wrote your first program!"

**Demo 2: Personalize It (3 min)**

```python
print("Hello, my name is [Your Name]!")
```

**Student Practice (2 min):**

- Students change the message to include their name
- Encourage creativity: pets, favorite things

**Demo 3: Multiple Lines (3 min)**

```python
print("Welcome to Python!")
print("I am learning to code!")
print("This is awesome!")
```

**Key Teaching Point:**

> "See how each `print()` creates a new line? Python reads from top to bottom, just like you read a book."

**Troubleshooting Tips for Instructors:**
| Problem | Likely Cause | Solution |
|---------|--------------|----------|
| Nothing happens | Didn't click run | Show where run button is |
| Red error message | Typo in `print` | Check spelling letter by letter |
| `SyntaxError` | Missing quote or parenthesis | Look for matching pairs |
| Student frozen | Overwhelmed | Have them clear all and start fresh with you |

## üéØ What is Python?

Have you ever wondered how Minecraft knows where to place a block? Or how YouTube picks videos you might like? The answer is **code** - and Python is one of the most popular ways to write it!

**Python** is a programming language that lets you give instructions to computers. Think of it like learning a new language - but instead of talking to people in Spanish or French, you're talking to computers in Python!

### Why "Python"?

Python was created in 1991 by a programmer named Guido van Rossum. He named it after a funny British TV show called "Monty Python's Flying Circus" - NOT after the snake! üé≠

### The Golden Rule of Coding

Computers are like very helpful robots that follow instructions EXACTLY. If you tell a robot "go over there," it won't know what you mean. But if you say "walk forward 10 steps, turn right, walk forward 5 steps" - now it can do it!

That's what coding is all about: giving clear, specific instructions.

## ‚ö†Ô∏è Common Mistakes (and How to Fix Them!)

Every programmer makes mistakes - even professionals! Here's how to spot and fix the most common ones:

### Mistake 1: Forgetting Quotation Marks

‚ùå **Wrong:**

```python
print(Hello World)
```

**Error:** Python gets confused! It thinks Hello and World are special commands.

‚úÖ **Right:**

```python
print("Hello World")
```

**Remember:** Text always needs quotation marks around it!

---

### Mistake 2: Wrong Spelling or Capitals

‚ùå **Wrong:**

```python
Print("Hello World")
```

**Error:** Python knows `print` but not `Print` - capital letters matter!

‚ùå **Also Wrong:**

```python
prnt("Hello World")
```

**Error:** Typos will break your code.

‚úÖ **Right:**

```python
print("Hello World")
```

**Remember:** `print` is always lowercase!

---

### Mistake 3: Missing Parenthesis or Quote

‚ùå **Wrong:**

```python
print("Hello World"
```

**Error:** Missing the closing `)` parenthesis!

‚ùå **Also Wrong:**

```python
print("Hello World)
```

**Error:** Opened with `"` but never closed it!

‚úÖ **Right:**

```python
print("Hello World")
```

**Remember:** Parentheses `()` and quotes `""` always come in pairs!

## üéì Key Things to Remember

| Rule                              | Example              |
| --------------------------------- | -------------------- |
| `print()` displays text on screen | `print("Hello!")`    |
| Text goes inside quotation marks  | `"Like this"`        |
| Parentheses come in pairs         | `print(` needs `)`   |
| Quotes come in pairs              | `"text"` not `"text` |
| Python is case-sensitive          | `print` not `Print`  |
| Spelling matters!                 | `print` not `prnt`   |
