# Kids Learn AI — Website Redesign Direction

## Goal

Create one recognizable product system that feels modern, credible, and welcoming without making every area of the website look identical.

## Current experience

The website currently has three competing visual modes:

- Public pages rely on long promotional sections, large gradients, and many feature cards.
- Student pages use colorful gradients, oversized cards, badges, and motion in many places at once.
- Admin pages are becoming denser and more operational.

The components are individually functional, but the hierarchy, spacing, color usage, and navigation density change too much between areas.

## Recommended direction: Bright Intelligence

A clean educational product with a calm neutral foundation, strong blue primary actions, and small warm accents. The brand should feel intelligent and optimistic rather than childish or corporate.

### Visual principles

1. Use color to communicate meaning, not to decorate every surface.
2. Use one clear primary action per section.
3. Prefer borders and tonal surfaces over large shadows and gradients.
4. Use tighter spacing inside tools and more breathing room on marketing pages.
5. Keep rounded corners moderate and consistent.
6. Make progress, next steps, and status immediately visible.

## Shared foundation

### Color

- Canvas: soft off-white or very light cool gray
- Surface: white
- Text: deep navy/slate
- Primary: confident blue
- Accent: warm amber/orange, used sparingly
- Success: green
- Warning: amber
- Error: red

Gradients should be reserved for a small number of brand moments, such as the homepage hero or student achievement states.

### Typography

- Use one modern sans-serif family throughout.
- Keep headings compact with tighter tracking.
- Use sentence case instead of title case for interface labels.
- Increase contrast between page titles, section headings, supporting copy, and metadata.

### Shape and elevation

- Default card radius: 10–12px
- Default control radius: 8px
- Pills only for statuses, filters, and compact categories
- Use subtle 1px borders as the default separation method
- Use shadows only for floating elements and important interactive states

### Spacing

- Base spacing unit: 4px
- Compact UI: 8–16px internal spacing
- Standard sections: 24–32px separation
- Marketing sections: 64–96px separation
- Avoid nested cards with padding at every level

## Experience-specific rules

### Public website

Audience: parents and prospective students.

- Lead with trust, outcomes, curriculum clarity, and a free-trial action.
- Shorten the homepage and remove repeated feature messaging.
- Use real student/project imagery purposefully.
- Keep pricing and enrollment steps visible and easy to compare.
- Simplify the main navigation; move secondary resources into grouped menus or the footer.

### Student experience

Audience: children and teens actively learning.

- Make the next lesson or next class the dominant element.
- Organize secondary content into clear sections instead of equal-sized cards.
- Preserve playful accents for progress, achievements, and interactive learning.
- Reduce decorative gradients and competing badges.
- Keep touch targets generous even while reducing unused space.

### Admin experience

Audience: instructors and operators.

- Favor information density, tables, filters, and clear work queues.
- Group navigation by task.
- Keep page headers and metric summaries compact.
- Surface exceptions and pending work before passive statistics.

## Information architecture

### Public navigation

- Programs
- How it works
- Projects
- Pricing
- Resources
- Sign in
- Book a free trial

### Student navigation

- Home
- Learn
- Practice
- AI labs
- Progress

### Admin navigation

- Workspace: overview, enrollments, schedules, submissions
- People: students, inquiries
- Content: lessons, quizzes, teacher notes
- Insights: analytics, concept labs, setup guide

## Recommended delivery order

1. Foundation: colors, typography, spacing, radii, buttons, cards, and page containers.
2. Shared navigation: simplify the public header and separate student navigation concerns.
3. Public conversion journey: homepage, pricing, inquiry, and trial booking.
4. Student core journey: dashboard, lessons catalog, and lesson viewer.
5. Admin system: apply the compact structure across all operational pages.
6. Secondary pages: blog, about, contact, settings, and utilities.

## Success criteria

- Visitors understand the offer and primary action within the first viewport.
- Students can identify their next activity without scanning the whole dashboard.
- Admin users can find pending work and primary tools quickly.
- Shared elements have consistent typography, spacing, radius, and interaction states.
- Mobile layouts retain hierarchy without turning every section into a long stack of cards.
